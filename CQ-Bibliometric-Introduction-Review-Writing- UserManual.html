CQ Introduction & Review Automated Writing Skill | User Manual

本系统支持从研究题目、话题或研究设想切入，构建结构化检索策略或直接导入本地资源，完成全量文献分析、系统综述生成及 SSCI 级引言撰写，输出具备数据追溯性的标准学术文档。

This framework automates academic literature synthesis starting from a research topic, query, or conceptual thought. It generates optimized query strategies, digests multi-source bibliographic metadata or local PDFs, executes formal modeling, and authors structured literature reviews or SSCI-grade introductions with rigorous source-to-claim tracking.

02 / Execution Modes

入口模式：按需选择数据源

1. API 在线自动检索

在检索策略获批后，调用 OpenAlex 引擎进行自动化文献拉取与精准筛选。此模式需配置对应密钥。

python scripts/review_pipeline.py search-strategy --task TASK --mode api-search

2. 无 API 本地资源导入

无需在线接口。系统生成标准化 Web of Science (WoS) 检索表达式供手动检索后，直接读取本地导出的题录或全文。

python scripts/review_pipeline.py search-strategy --task TASK --mode strategy-only
python scripts/review_pipeline.py ingest --task TASK /path/to/ris-folder
python scripts/review_pipeline.py extract --task TASK /path/to/pdf-folder

语料精细化调控

数据导入完成后，您可自主进行三项高阶设置：(1) 限定为核心样本或全量分析；(2) 显式指定学术缺口 (Research Gap) 的演进路线；(3) 针对系统综述与 SSCI 引言分别定制差异化的逻辑结构。

Execution Modes: Online Search vs. Local Ingestion

1. API-Driven Retrieval

Queries the OpenAlex database directly after the search strategy is approved. This mode automates data extraction but requires valid API credentials.

python scripts/review_pipeline.py search-strategy --task TASK --mode api-search

2. Local Bibliographic Ingestion

Bypasses API connections. Generates optimized Web of Science (WoS) search strings for manual execution, then ingests exported files or PDFs recursively.

python scripts/review_pipeline.py search-strategy --task TASK --mode strategy-only
python scripts/review_pipeline.py ingest --task TASK /path/to/ris-folder
python scripts/review_pipeline.py extract --task TASK /path/to/pdf-folder

Granular Corpus Control

Following data import, three configuration gates are available: (1) Define core vs. extended corpus filters; (2) Instruct specific research gap trajectories; (3) Independently tune structural parameters for both the comprehensive Review and the concise SSCI Introduction.

03 / Core Principles

核心原理：开箱前应知悉的 6 项设定

1

独立任务沙盒

每个研究项目均隔离于独立的 review-tasks/任务名 目录，确保中间元数据可完整追溯或随时断点续跑。

2

双语语义转化

输入中文课题后，底层架构会自动映射其对应的英文核心术语，保障国际检索式的构建质量。

3

证据层级划分

系统自动区分摘要级证据与全文级证据。仅包含摘要时使用弱断言，配有 PDF/DOCX 时则提取带有精确页码锚点的强断言。

4

异构文献级合并

支持将 OpenAlex 在线结果与手动导出的 WoS、知网、EndNote 等文献在字段级精准对齐并去重合并。

5

计量信号驱动

主题演进、突现术语、网络中心性等均为计量特征。系统要求其必须与叙事性文献证据卡相结合进行归纳解释。

6

断言追踪账本

综述中的每一句结论都会绑定唯一的 Claim ID 和 Record ID，生成完全一致、不含冗余引用的 APA 7 参考文献表。

Core Principles: Six Operational Orientations

1

Sandbox Workspace

Every project uses an isolated directory under review-tasks/task-id to preserve all states, raw responses, and pipeline logs.

2

Bilingual Cross-Mapping

Native language inputs are translated and mapped to established domain taxonomies to maximize overseas search precision.

3

Evidence Layering

Differentiates abstract-level records from full-text records. Claims based on partial data default to safe hedging language.

4

Multi-Engine Merge

Merges metadata from OpenAlex, WoS, CNKI, EndNote, and custom RIS targets at the attribute level with deduplication tracking.

5

Structural Inferences

Bibliometric parameters (NMF weights, Betweenness, Burst Strength) serve as diagnostic indicators, bound to source text snippets.

6

Deterministic Auditing

Every single analytical claim cross-references a specific Claim Ledger entry to completely eliminate synthetic citations.

06 / Search & Credentials

检索体系、交叉翻译与鉴权凭据

双语跨系统对齐原理

输入包含中文时，系统会自动构造学术翻译实体：包含 title_zh、title_en、双语交叉概念簇，并强制对齐中文核心检索词与英文高级检索表达式。后台脚本会强制校验英文命名的规范性，缺少任意语料层级时将终止执行。

确认方案时的审查核心

  • 自变量、因变量、中介/调节变量及理论对象是否已全部覆盖
  • 同义词包是否匹配主流期刊（如 SSCI/CSSCI）的高频表达
  • 排除性术语是否设置得过于宽泛，导致“误伤关键关联文献”
  • 时间跨度、文献源分类（如 Article/Review）是否符合发表目标

API 密钥配置流程

数据隐私安全防护： 切勿直接将敏感密钥以聊天文本发送。系统在执行检索需要 OpenAlex Key 时，会自动拉起本地安全配置面板。密钥通过操作系统底层安全存储区（如 macOS Keychain 或 Windows Credential Manager）进行隔离保存。 
# 凭据管理指令集
python scripts/review_pipeline.py credentials setup
python scripts/review_pipeline.py credentials status
python scripts/review_pipeline.py credentials test

Search Syntax, Alignment, & Secure Credentials

Cross-Language Taxon Mapping

When processing mixed-language titles, the host runtime creates structured models containing title_zh, title_en, and synchronized query components. Strict schemas prevent submission if English field targets contain multi-byte syntax, keeping search boundaries sound.

Search Audit Checklist

  • Ensure predictors, criteria, mediators, and theoretical concepts are fully captured.
  • Verify alternate synonyms align with dominant indexing vocabularies in top journals.
  • Confirm boolean NOT conditions do not exclude relevant hybrid domains.
  • Ensure publication windows and document class definitions align with research scope.

Credential Encryption Matrix

Security Protocol: Never dump API keys into active chat blocks. If an execution route lacks key context, it calls standard interactive shells. Keys route securely to native credential stores (e.g., macOS Keychain).

07 / Literature Ingestion

多源题录数据融合

数据解构引擎原生兼容 RIS、ENW、NBIB、BibTeX、WoS 纯文本、知网 CSV、以及自定义 Excel 等结构。通过路径传递，系统将对输入流运行清洗对齐。

去重冲突判别树

基于全局 DOI 强锁定 → OpenAlex ID/PMID 匹配 → “标准化文献题名 + 出版年份 + 第一作者姓氏”的相似度模糊计算进行二次去重。

多维合并而非简单覆盖

系统拒绝暴力丢弃数据，而是自动执行交叉补全：对摘要、关键词字段进行智能互补，并独立保留各数据源的被引频次与检索线索。

数据审计重点产物

  • 02_corpus/corpus.jsonl：全任务唯一的权威对齐语料主库。
  • 07_logs/ingest_report.json：记录总数、有效增量数、去重合并数及异常项。
  • 02_corpus/records.xlsx：专为人工快速浏览与二次复核生成的电子表格。

Bibliographic Data Ingestion

The system digests raw RIS, ENW, NBIB, BibTeX, WoS tagged strings, CNKI tables, and customized XLSX structures natively. Simply supply the directory or file reference.

Resolution Ordering

DOI strictly dominates -> followed by unique engine keys (PMID/OpenAlex UID) -> falling back to fallback fuzzy matching ("Levenshtein Title + Year + Author Surname").

Non-Destructive Properties

Instead of deletion, the pipeline aggregates attribute overlaps. It retains distinct citation rankings and tracks multi-source database lineage perfectly.

Core Data Manifests

  • 02_corpus/corpus.jsonl: The master relational datastore representing the task's source of truth.
  • 07_logs/ingest_report.json: Balance audits summarizing input density, deduplicated entries, and warning labels.
  • 02_corpus/records.xlsx: Formatted spreadsheets optimized for rapid manual curation and filtering.

09 / Bibliometric Modeling

科学计量算法架构与语义矩阵构建

Nordic NMF 主题分类模型

基于文献名与摘要流生成 TF-IDF 特征，在 4-8 个维度区间内自动计算最佳的主题空间大小。剔除外部噪声数据干扰。

KMeans 异质性结构审视

充当结构校验层。仅当聚类效果和最小分类群规模达到置信阈值，且具备增量信息时，才会转化为结构论断证据。

SNA 社会网络与拓扑漏洞

计算共同作者、高频词共现、文献共引及耦合网络。计算 PageRank、介数中心性、Burt 结构洞约束值及有效规模。

突现术语与演化演进

利用年度权重变动序列捕捉长尾高强短语与引文爆发现象，清晰输出突现时间起点、峰值及背后的核心支撑群。

数据解读原则： 计量模型输出的“拓扑指标”、“中心节点”属于结构层信号，在正式写入综述时，必须通过系统内置的 supporting_documents 交叉核验实际文献。

Mathematical Bibliometrics & Semantic Mining

NMF Topic Extraction

Transforms terms into TF-IDF representations, sweeping a k-range of 4–8 to select topic distributions via coherence and stability testing.

KMeans Structural Diagnostics

Acts as an outlier validator. Suppresses noisy clusters unless silhouette scores demonstrate information gain beyond linear models.

SNA Topological Analytics

Constructs co-authorship, keyword, and co-citation webs. Extracts structural metrics: Betweenness, PageRank, and Burt's structural hole constraints.

Burst Analysis

Tracks velocity vectors over time series to map keyword usage shifts, generating onset, intensity peaks, and references.

Usage Warning: Graph centralities indicate structural significance only. Writing claims requires evaluating localized text elements within the evidence package.
10 / Academic Writing

深度实证：从元数据到标准学术文本

四层证据链驱动架构

  1. 个案文献证据卡：深度结构化抽取单篇文献的研究假设、理论底座、因果路径、核心样本、关键结论与局限性锚点。
  2. 聚合主题简报 (Dossier)：将孤立的证据卡组织为面向特定主题、研究演进、冲突视角的结构化知识库。
  3. 论断追踪账本 (Claim Ledger)：建立可追溯的论断矩阵。记录每条核心学术断言的证据级别、正反向论证力度与反例分布。
  4. 大纲映射分级索引：精确控制大纲各小节应调阅的证据池，防止模型在长文本生成中出现事实偏离。

1. 结构化系统综述

采用严谨的标准学术分节结构：研究边界与现状、演进脉络、因果核心要素、核心变量机制、方法论视角、学界争议、前沿研究缺口。若核心语料库不大于 100 篇，文本引用的整体覆盖率可保障在 60% 以上。

2. SSCI 漏斗型绪论引言

专为高水平期刊打造。全文设计为由 8-12 个连续自然段构成的贯通性论述，**不含任何列表块与显式小标题**。行文完全遵循标准学术漏斗逻辑：探讨宏观战略背景 → 聚焦特定研究域 → 解构关键冲突并确立核心研究缺口 → 顺势推出本文研究目的 → 阐明独特的理论创新与边际贡献。

无网络依赖的单文件 HTML 交付

系统最终交付包含完整格式的单文件 HTML 报告。在浏览器内点击“导出 Word”将实时无损封装生成标准 .docx 文档；点击“导出 PDF”则完美调用 A4 打印样式排版，避免任何格式错乱。

Evidence-to-Prose Synthesis

Four-Tier Evidence Framework

  1. Individual Evidence Cards: Maps variables, contexts, mechanisms, and findings directly back to resource text fragments.
  2. Aggregated Dossiers: Groups semantic nodes into themes, tracking network features and chronological trajectories.
  3. Claim Ledger: Tracks systematic propositions, auditing empirical direction, sample weights, and counter-evidence blocks.
  4. Section Allocation Indexes: Injects deterministic content maps into the generative context window to control topical coverage.

1. Comprehensive Literature Review

Deploys a canonical subsections layout: Boundaries, Timeline, Causal Engines, Methodology Shifts, and Domain Gaps. Assures at least 60% citation coverage for workspaces with under 100 source records.

2. SSCI Funnel-Style Introduction

Designed for premium journal submissions. Emits a clean stream of 8–12 highly coherent paragraphs **without bullet lists or h3 headers**. Followes the classic academic funnel: Broad Context -> Targeted Domain Domain -> Theoretical Friction and Gap Isolation -> Statement of Purpose -> Theoretical and Practical Contributions.

Zero-Dependency Standalone HTML

The system exports a self-contained HTML asset. Triggering "Export to Word" packages a structured .docx file completely client-side. Triggering "Export to PDF" formats the document perfectly to A4 print standards.

13 / Quality Audit

高标准交付规范检验指标

  • 自动化审计文件 validation_report.json 处的 valid 指标必须呈现为 true。
  • 中英文高级交叉检索式语义完备，且后台日志中留有用户明确输入的“批准”日志印记。
  • 核心语料库经过严格的情境净化，未混入医疗、儿童教育等非直接相关的常识性领域文献。
  • NMF 主题建模聚类词汇具备高度的营销或管理学学术可解释性，彻底剔成了单纯由系统后台软件或基础理学派生的元数据噪音（如 physics 等干扰词）。
  • 正文断言与文献证据完全对齐，严禁出现没有任何事实文献支撑的“孤立推论章节”。
  • 若使用的是摘要级语料，行文应严谨采用“摘要揭示”、“现有文献表明”等限制性修饰词。对学术缺口（Research Gap）的定性表述必须采用“在当前检索语料中较少出现”，绝对禁止出现“从未有人研究”等违反科学常识的绝对化表述。

Rigorous Acceptance Standards

  • The automated diagnostic field valid within validation_report.json must report strictly as true.
  • Query blocks must capture bilingual criteria and carry signed approval codes in the pipeline manifest.
  • Target thematic pools must be clean and free from domain confusion (e.g., medical or educational noise).
  • NMF thematic profiles must present sharp domain semantic differentiation, completely free from database metadata artifacts (such as physics or engineering keywords).
  • Every interpretive section must balance across core cluster densities, avoiding orphaned assertions unsupported by source data.
  • Abstract-only synthesis must use hedged phrases ("The summarized records imply..."). Deficits in research depth must be stated as "rarely observed in the sampled data" rather than sweeping claims like "no prior research exists".
14 / Troubleshooting

高频故障精准排除路线

系统抛出异常：“Missing OpenAlex API Key”

请指示终端工具执行以下指令：python scripts/review_pipeline.py credentials update --name OPENALEX_API_KEY。在随后弹出的系统安全交互输入窗口中粘贴您的凭据密钥。切勿直接将其粘贴在聊天对话文本中。

系统检索阻塞：中文课题无法激活在线搜索

这表明当前的方案校验器未通过。请要求大语言模型：“补齐缺失的英文标题学术翻译、完善交叉概念组，并重新触发 validate-plan”，切勿强行绕过验证流程。

检索数据量过于宽泛，主题演进混杂不清

不要在主任务中反复折腾。请立即通过 focus 命令切分出独立的派生聚焦任务区间。显式限定核心纳入边界、精炼理论底座条件，并重新计算 NMF 主题模型。

文内有引用标签 却缺少文末参考文献表

此现象属于纯粹的引文同步未刷新。请在根路径直接敲入指令：python scripts/review_pipeline.py sync-references --task TASK --document review，系统将基于正文真实标签一键重构标准的参考文献附表。

Troubleshooting Matrix

System error throws: "Missing OpenAlex API Key"

Instruct the interface to execute: python scripts/review_pipeline.py credentials update --name OPENALEX_API_KEY. Use the secure interactive window to save your credential. Do not dump keys directly in chat feeds.

Search validation aborts multi-language operational setup

This implies structural values failed standard sanity checks. Command the agent to: "Repair bilingual taxon fields, synchronize concept sets, and run validate-plan again" to reset the state machine cleanly.

Result counts are excessive or thematic indicators bleed across scopes

Avoid editing raw master tables. Use the focus routine to slice out a refined subset workspace. Apply tighter inclusion bounds, and re-run NMF topic calculations.

In-text tokens `` display correctly but back-end list fails to render

The citation registry has not been synchronized. Execute the sync loop directly from your workspace console: python scripts/review_pipeline.py sync-references --task TASK --document review to align the files.
