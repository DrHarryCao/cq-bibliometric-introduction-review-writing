## 独立理论库

理论库位于技能目录之外，默认使用平台数据目录，也可由`CQ_THEORY_HOME`覆盖。技能包不内置具体理论。用户可在没有研究任务、API或语料时单独运行`theory-library init|ingest|verify|promote|search|show|update|disable|export|import`。

导入HTML、PDF、DOCX、DOC、TXT或Markdown时，只自动建立候选卡。默认登记原文件位置和哈希，不复制原始资料；只有显式使用`--copy-sources`才复制到外部用户理论库。网页介绍只是理论与来源线索，不能自动成为权威理论陈述。

## 理论卡核验

读取`assets/theory_card.template.json`。宿主必须核对理论的核心构念、机制、假设、边界、竞争解释及奠基文献或权威综述。

- `candidate`：只发现线索。
- `source-located`：宿主已完成字段和来源核对，等待用户显式晋升。
- `content-verified`：用户确认后可作为已核验背景知识。
- `disabled`：停用或存在无法解决的冲突。

不要绕过付费墙。无法获得来源时保持候选状态。理论库不是当前研究的经验性证据。

## 任务检查点

仅在解释A级/B级缺口、比较机制、构建模型/命题或撰写理论贡献时询问理论支持模式：

- `combined`：本地理论库与宿主LLM候选。
- `local-only`：只读取本地库。
- `llm-only`：宿主提出候选并核验来源。
- `skip`：直接使用任务证据写作。

选择后运行`theory-recommend --phase prepare`。宿主补全`theory_fit_ledger.jsonl`后运行`compile`和`validate`。默认最多比较8个候选，正式模型优先采用1–3个确有解释增量的理论。

## 理论适配

每个采用决定都要检查：解释对象、分析层级、核心假设、构念映射、机制、边界条件、竞争理论、可区分预测、当前任务文献及权威来源。关键词相似不等于适配。LLM记忆不等于来源核验。

映射顺序为：

`知识失效 → 理论假设 → 构念 → 机制 → 边界 → 竞争解释 → 可区分预测 → 当前证据 → 研究设计`

## 自动降级

理论支持永不阻断主写作流程。库为空、文件损坏、索引失败、无匹配理论、来源未核验或理论冲突时：

1. 跳过损坏条目并保留可用条目。
2. 排除未核验的确定性理论陈述。
3. 把必要但部分支持的关系写成明确待检验命题。
4. 改用任务文献支持的“机制—边界—竞争解释”理论中性叙事。
5. 删除非必要且无法支持的理论性句子。
6. 继续完成综述、绪论、参考文献、RIS和HTML。

理论模块只允许`validated|exploratory|skipped|degraded|unavailable`，不得把任务设为`blocked`。技术失败写入`theory_degradation_log`并自动续跑。

## 写作边界

- 理论不足不能被宣称为Research Gap。
- 一般理论命题不能冒充当前情境下的经验结论。
- 正文采用理论时应登记奠基或权威来源。
- 理论适配不足时自动改写或删除相关句，不停止整个交付。
- 理论模块的非阻断性不能放宽虚假引用、未知文献、因果夸大、原子论断支持或文内—文末—RIS一致性标准。
