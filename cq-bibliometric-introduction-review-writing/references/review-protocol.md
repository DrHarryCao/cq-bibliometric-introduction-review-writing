# 中文系统性综述写作与审计

## 写作顺序

1. 先运行 `writing-brief --document review`，记录用户想突出的缺口方向；用户跳过时执行全维度缺口审查。
2. 运行`build-gaps`完成三级缺口账本，再运行`recommend-design`；没有A/B级缺口时只写降级研究机会并保留警告。
3. 从 `evidence_index.md` 选择当前章节，只读取相关 dossier、claim ledger 和代表性卡片。
4. 为本节建立“研究问题—共识—分歧/反例—机制—情境边界—过渡”叙事契约和“原子论断—对象/机制/结果—支持文献—限定词”小表。原子论断是不能再拆而仍保持独立可验证意义的最小陈述。
5. 写作时使用 APA 7 可见引用，并在每个原子论断之后保留 `[cite:R...,R...]` 审计绑定；每节立即运行 `audit-writing --source FILE --scope SECTION_ID`，清空 `citation_repair_queue.jsonl` 与 `citation_attribution_queue.jsonl` 后再合并。禁止把段末全部引文机械复制到每句，也禁止用一个大型引文组同时支持不同对象、机制、媒介和结果。
6. 新增证据必须进入其所支持的理论、机制、争议、边界或方法段落；禁止在正文末尾新增“扩展主题证据”“补充引用”等孤立章节来凑引用覆盖。
7. 合并后运行 `sync-references --document review`，以正文实际使用的审计绑定自动生成文末参考文献。
8. 获得提纲或写作简报确认后，自动完成所有分节、缺口审计、动态设计匹配、补足引用、合并、同步和验证；不在中间进度节点停止。
9. 同步参考文献后运行 `audit-writing --document review`，对每个事实性句子完成语义支持判定；引用出现但不支持该句仍属失败。`supported`必须同时有证据ID、证据等级、支持范围和审计说明，不允许只批量改状态。
10. 生成明确标记的HTML草稿预览，暂停等待用户“确认综述”或提出修改意见。只有 `document-approval --document review --status approved` 且正文哈希未变时，才进入绪论偏好。
11. 证据提示版通过后，运行 `build-publication prepare`。投稿版删除“聚焦语料、摘要证据、直接证据、证据等级、当前任务”等内部流程语言；`supported`论断可直接保留，核心`partial`论断必须改写成“本研究提出/有待检验”的命题，`unsupported`论断必须删除或继续阻止交付。宿主对改写句的逐条判断写入`publication_support_reviews.jsonl`，至少记录句子ID、支持状态、设计适配、证据层级、论断范围、独立样本数和审计说明；禁止批量把`pending`改成`supported`。完成后运行`compile`与`validate`。

## 引用密度与主题论述

- 每个知识主题的实质性段落都必须引用该主题相关文献，并同时覆盖基础、代表、新近和反向/边界证据。
- 凡出现“一些研究”“既有文献”“研究表明/发现/指出”“语料显示/呈现”“证据表明”等句式，同句必须有可见引用和审计绑定。
- 强综合性论断至少使用3项独立研究，不得用不相关文献、裸record ID列表或文末堆砌满足配额。
- 并列列出多个研究对象、技术、机制或结果时，必须逐项或按同质子组配置引文。例如历史重演、文化表演和数字叙事若由不同文献支持，应拆成独立原子论断，不能共享句末引文串。

## 默认结构

研究范围与语料边界；发展阶段与主题结构；理论/概念/机制；方法与情境；争议与边界条件；经校准的研究空白；未来议程。方法说明要报告数据库、检索日、查询边界、去重、全文覆盖和局限。

## 独立 SSCI 绪论

用户确认综述后，才单独询问绪论希望突出的缺口或研究问题，并运行 `writing-brief --document introduction`。接受一个/多个方向、“跳过”或“不写绪论”，不默认继承综述偏好。绪论与带小标题的系统综述分开生成。默认中文 3,000–5,000 字、8–12 个连续正文自然段，正文无小标题、列表或表格，文末必须有自动同步的参考文献。审计版用 `[role:...]` 为每段登记现实重要性、情境收束、进展、矛盾、机制竞争、边界、缺口、目标和贡献角色；清洁版删除角色与内部ID。第一段必须来自 `social_context_sources.jsonl` 中可核验的官方/权威现实背景。最后运行 `audit-writing --document introduction`。

## 表述纪律

- “语料中呈现”用于计量结果；“研究发现”仅用于内容证据。
- 摘要证据写“摘要报告/表明”，不声称核查了全文方法。
- gap 使用“现有语料尚少见/证据不足以判断”，避免“从未研究”。
- “研究较少/换情境/未使用某方法”只能写成C级机会；核心gap必须指出当前解释或可信推断如何失败。方法先由研究问题与设计决定，不能默认PLS-SEM、ANN、fsQCA或任何其他组合。
- 相关不写成因果；单一情境不外推普遍规律；高被引不等于高质量。
- 投稿正文不显示证据等级或任务流程提示，但不得因此提高论断确定性。弱证据的核心关系改写为待检验命题，非核心关系删除；正式方法部分仍保留数据库、筛选和纳入数量等可复现信息。

## 最终检查

核对每个事实性段落的 claim ID、APA 文献是否存在、全文锚点是否可打开、引用覆盖与主题配额、每节证据是否分布合理、正文标题是否来自已批准提纲、同一来源是否被过度依赖、反例是否遗漏，以及正文引用集合与文末参考文献集合是否完全一致。交付时删除内部 claim ID 之前，保留一份含 ID 的审计稿。
