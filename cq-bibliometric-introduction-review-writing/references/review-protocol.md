# 中文系统性综述写作与审计

## 写作顺序

1. 先运行 `writing-brief --document review`，记录用户想突出的缺口方向；用户跳过时执行全维度缺口审查。
2. 运行`build-gaps`完成三级缺口账本，再运行`recommend-design`；没有A/B级缺口时只写降级研究机会并保留警告。
3. 从 `evidence_index.md` 选择当前章节，只读取相关 dossier、claim ledger 和代表性卡片。
4. 为本节建立“研究问题—共识—分歧/反例—机制—情境边界—过渡”叙事契约和“论断—证据—限定词”小表。
5. 写作时使用 APA 7 可见引用，并在引文后保留 `[cite:R...,R...]` 审计绑定；分节检查后再合并。
6. 新增证据必须进入其所支持的理论、机制、争议、边界或方法段落；禁止在正文末尾新增“扩展主题证据”“补充引用”等孤立章节来凑引用覆盖。
7. 合并后运行 `sync-references --document review`，以正文实际使用的审计绑定自动生成文末参考文献。
8. 获得提纲或写作简报确认后，自动完成所有分节、缺口审计、动态设计匹配、补足引用、合并、同步和验证；不在中间进度节点停止。

## 引用密度与主题论述

- 每个知识主题的实质性段落都必须引用该主题相关文献，并同时覆盖基础、代表、新近和反向/边界证据。
- 凡出现“一些研究”“既有文献”“研究表明/发现/指出”“语料显示/呈现”“证据表明”等句式，同句必须有可见引用和审计绑定。
- 强综合性论断至少使用3项独立研究，不得用不相关文献、裸record ID列表或文末堆砌满足配额。

## 默认结构

研究范围与语料边界；发展阶段与主题结构；理论/概念/机制；方法与情境；争议与边界条件；经校准的研究空白；未来议程。方法说明要报告数据库、检索日、查询边界、去重、全文覆盖和局限。

## 独立 SSCI 绪论

综述完成后才询问是否继续，并运行 `writing-brief --document introduction`。绪论与带小标题的系统综述分开生成。默认中文 3,000–5,000 字、8–12 个连续正文自然段，正文无小标题、列表或表格，文末必须有自动同步的参考文献。第一段必须来自 `social_context_sources.jsonl` 中可核验的官方/权威现实背景。审计版保留 cite/source ID，清洁版不得含内部 ID。

## 表述纪律

- “语料中呈现”用于计量结果；“研究发现”仅用于内容证据。
- 摘要证据写“摘要报告/表明”，不声称核查了全文方法。
- gap 使用“现有语料尚少见/证据不足以判断”，避免“从未研究”。
- “研究较少/换情境/未使用某方法”只能写成C级机会；核心gap必须指出当前解释或可信推断如何失败。方法先由研究问题与设计决定，不能默认PLS-SEM、ANN、fsQCA或任何其他组合。
- 相关不写成因果；单一情境不外推普遍规律；高被引不等于高质量。

## 最终检查

核对每个事实性段落的 claim ID、APA 文献是否存在、全文锚点是否可打开、引用覆盖与主题配额、每节证据是否分布合理、正文标题是否来自已批准提纲、同一来源是否被过度依赖、反例是否遗漏，以及正文引用集合与文末参考文献集合是否完全一致。交付时删除内部 claim ID 之前，保留一份含 ID 的审计稿。
